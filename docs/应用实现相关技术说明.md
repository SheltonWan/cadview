## 1. 技术栈总览

| 层次 | 技术 | 版本要求 |
|------|------|----------|
| 移动/桌面原生宿主 | Flutter / Dart | SDK ^3.11.0 |
| CAD 查看器前端 | Vue 3 + TypeScript | Vue ^3.5 |
| CAD 解析与渲染 | @mlightcad/cad-viewer | ^1.4.13 |
| 前端构建工具 | Vite | ^8.0 |
| 内嵌 HTTP 服务 | Dart shelf | ^1.4.2 |
| WebView 容器 | webview_flutter | ^4.10.0 |

整体思路：Flutter 负责原生 UI 壳和文件系统访问；Web 技术（Vue + Canvas/WebGL）负责 CAD 图纸的解析与渲染；两者通过**内嵌 HTTP 服务 + JavaScript Channel** 衔接。

## 2. Flutter 原生层实现

### 2.1 入口与主题（`lib/main.dart`）

应用入口调用 `WidgetsFlutterBinding.ensureInitialized()` 确保平台绑定初始化完毕，随后启动 `PropOsCadApp`。

全局 `ThemeData` 基于 Material 3 暗色方案，主色调为 `#0078D4`（Windows 蓝），针对以下组件做了精细化 `WidgetState` 颜色覆盖：

- `IconButtonThemeData`：hover 15% 蓝色叠加 / pressed 30% 蓝色叠加
- `ElevatedButtonThemeData`：hover / pressed 白色叠加
- `OutlinedButtonThemeData`：hover / pressed 白色叠加
- `TextButtonThemeData`：前景色浅蓝 `#4FC3F7`，hover / pressed 叠加
- `ListTileThemeData`：选中色 `#4FC3F7`，hover 蓝色叠加

### 2.2 文件浏览界面（`lib/screens/file_browser_screen.dart`）

`FileBrowserScreen` 为有状态组件，维护以下状态：

| 字段 | 类型 | 说明 |
|------|------|------|
| `_currentDir` | `Directory?` | 当前已选目录，`null` 时显示欢迎页 |
| `_dwgFiles` | `List<FileSystemEntity>` | 过滤后的 DWG/DXF 文件列表 |
| `_loading` | `bool` | 是否正在执行文件选择操作 |

**文件夹扫描逻辑（`_loadFilesFromDir`）：**

```dart
final entities = dir.listSync(followLinks: false);
final dwg = entities
    .whereType<File>()
    .where((f) {
      final ext = p.extension(f.path).toLowerCase();
      return ext == '.dwg' || ext == '.dxf';
    })
    .toList()
  ..sort((a, b) => p.basename(a.path).compareTo(p.basename(b.path)));
```

- 使用 `listSync(followLinks: false)` 枚举目录第一层，不跟随符号链接，不递归子目录。
- 以扩展名大小写无关方式匹配 `.dwg` / `.dxf`。
- 文件列表按文件名字母顺序升序排列。

**文件选取方式：**

- `_pickFolder()`：调用 `FilePicker.platform.getDirectoryPath()` 唤起系统目录选择对话框，获得路径后调用 `_loadFilesFromDir`。
- `_pickFile()`：调用 `FilePicker.platform.pickFiles()` 并指定 `allowedExtensions: ['dwg','dxf','DWG','DXF']`，选中后直接跳转 `ViewerScreen`。

### 2.3 CAD 查看界面（`lib/screens/viewer_screen.dart`）

`ViewerScreen` 接收 `filePath`（本地文件绝对路径）作为必传参数，在 `initState` 阶段执行异步初始化：

1. **启动 `LocalServer`**（单例，已运行则跳过）。
2. **构建 `WebViewController`**：
   - 启用无限制 JavaScript（`JavaScriptMode.unrestricted`）。
   - 注册名为 `FlutterBridge` 的 JavaScript Channel，接收来自网页的消息。
   - 设置 `NavigationDelegate`：`onPageFinished` 时调用 `_loadFile()`。
3. **加载查看器页面**：`http://127.0.0.1:{port}/viewer/`。

**`_loadFile()` 实现：**

```dart
final encodedPath = Uri.encodeComponent(widget.filePath);
final fileUrl = 'http://127.0.0.1:$port/dwg?path=$encodedPath';
controller.runJavaScript(
  "if (window.cadViewer) { window.cadViewer.loadFile('$fileUrl'); }",
);
```

文件路径经 `Uri.encodeComponent` 编码后作为查询参数传递，通过 `runJavaScript` 调用网页暴露的 `window.cadViewer.loadFile()` 接口。

**界面状态机：**

| 状态 | 显示内容 |
|------|----------|
| `_controller == null` | 加载动画（"正在启动查看器…"） |
| `_controller != null && !_viewerReady` | WebView 已渲染，叠加加载蒙层（"正在加载 CAD 文件…"） |
| `_viewerReady == true` | 完整显示 CAD 图纸，加载蒙层消失 |
| `_serverError == true` | 错误提示页面 |

## 3. 本地 HTTP 服务（LocalServer）

文件：`lib/services/local_server.dart`

### 3.1 设计目标

浏览器安全策略（CORS、SharedArrayBuffer）要求 CAD 查看器的 WebAssembly 模块运行在同源的 HTTP 服务下，而非直接通过 `file://` 协议加载。因此内嵌一个轻量 HTTP 服务，将 Vue 前端资源和本地 DWG 文件都通过 `http://127.0.0.1` 提供给 WebView。

### 3.2 单例生命周期

```dart
static final LocalServer instance = LocalServer._();
```

使用 Dart 单例模式，确保全局只有一个 HTTP 服务实例。`start()` 具有幂等性：已运行时立即返回，不重复绑定端口。

### 3.3 端口分配

```dart
_server = await shelf_io.serve(handler, InternetAddress.loopbackIPv4, 0);
_port = _server!.port;
```

监听地址为 `127.0.0.1`（IPv4 回环），端口传入 `0` 让操作系统自动分配空闲端口，避免端口冲突。

### 3.4 路由逻辑

| 路径 | 处理器 | 说明 |
|------|--------|------|
| `/dwg?path=<encoded>` | `_serveDwg` | 从本地磁盘读取 DWG/DXF 字节并返回 |
| `/viewer/**` | `_serveViewer` | 返回前端静态资源（HTML/JS/CSS） |

### 3.5 `/dwg` 处理器安全机制

- 校验 `path` 查询参数非空。
- 使用 `File.existsSync()` 验证文件存在性，不存在则返回 404。
- 读取字节直接响应，不执行任何脚本或模板渲染，不存在路径遍历注入风险（路径来自本应用内部的文件选择对话框）。

### 3.6 静态资源处理器

Viewer 资源预先写入临时目录，服务器从该目录按相对路径查找文件。请求中的 `viewer/` 前缀被剥离，默认文件为 `index.html`。

### 3.7 安全响应头

所有静态资源响应均携带以下安全头，以支持 WebAssembly 多线程（SharedArrayBuffer 依赖）：

```
Cross-Origin-Opener-Policy: same-origin
Cross-Origin-Embedder-Policy: require-corp
```

### 3.8 CORS 中间件

全局 CORS 中间件为所有响应追加：

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, OPTIONS
Access-Control-Allow-Headers: *
```

OPTIONS 预检请求直接以 200 状态码短路返回。

## 4. CAD 查看器前端（cad_viewer_builder）

目录：`cad_viewer_builder/`

### 4.1 技术选型

| 技术 | 用途 |
|------|------|
| Vue 3 + Composition API | 前端框架，声明式 UI |
| TypeScript | 类型安全 |
| @mlightcad/cad-viewer | CAD 文件解析与渲染核心（含 WebAssembly LibreDWG 解析器） |
| Element Plus | UI 组件库（用于错误提示等） |
| vue-i18n | 国际化，当前设定 `locale='zh'` |
| Vite | 构建打包，输出至 `assets/viewer/` |

### 4.2 构建产物

Vite 将前端代码打包为以下文件，之后手动复制至 `assets/viewer/` 作为 Flutter asset：

```
assets/viewer/
├── index.html
└── assets/
    ├── index.js               # 主 bundle（含 Vue、Element Plus）
    ├── index.css              # 样式
    └── libredwg-parser-worker.js  # WebAssembly DWG 解析 Worker
```

### 4.3 核心组件（`src/App.vue`）

`App.vue` 使用 `<MlCadViewer>` 组件（来自 `@mlightcad/cad-viewer`），传入属性：

| 属性 | 值 | 说明 |
|------|-----|------|
| `:url` | `fileUrl`（响应式 ref） | 指向本地 HTTP 服务的文件 URL |
| `:locale` | `'zh'` | 界面语言为中文 |
| `:theme` | `'dark'` | 暗色主题 |

### 4.4 初始化流程（`onMounted`）

```typescript
onMounted(() => {
  (window as any).cadViewer = {
    loadFile(url: string) { fileUrl.value = url },
    closeFile()          { fileUrl.value = undefined }
  }
  notifyFlutter({ type: 'ready' })
})
```

Vue 组件挂载完成后：
1. 在 `window.cadViewer` 上挂载 API 供 Flutter 的 `runJavaScript` 调用。
2. 通过 `FlutterBridge.postMessage` 向 Flutter 发送 `ready` 消息，通知可以发送文件 URL。

## 5. Flutter ↔ JavaScript 通信桥

### 5.1 Flutter → JavaScript（命令下发）

Flutter 使用 `WebViewController.runJavaScript()` 执行 JavaScript，直接调用前端挂载在 `window.cadViewer` 上的方法：

| 调用 | 触发时机 |
|------|----------|
| `window.cadViewer.loadFile(url)` | 查看器页面加载完成后，或用户点击刷新按钮 |
| `window.cadViewer.closeFile()` | （预留，当前版本由页面导航销毁实现关闭） |

### 5.2 JavaScript → Flutter（事件上报）

前端通过 `FlutterBridge.postMessage(JSON.stringify(payload))` 向 Flutter 发送 JSON 消息：

| `type` | `payload` | Flutter 处理 |
|--------|-----------|--------------|
| `ready` | — | 设置 `_viewerReady = true`，触发 `_loadFile()` |
| `error` | 错误描述字符串 | 弹出红色 SnackBar 提示 |

Flutter 侧在 `_onBridgeMessage` 中解析消息：

```dart
void _onBridgeMessage(JavaScriptMessage message) {
  final data = jsonDecode(message.message) as Map<String, dynamic>;
  final type = data['type'] as String?;
  if (type == 'ready') {
    setState(() => _viewerReady = true);
    _loadFile();
  } else if (type == 'error') { ... }
}
```

## 6. 资源打包与提取机制

### 6.1 打包

在 `pubspec.yaml` 中声明：

```yaml
assets:
  - assets/viewer/
  - assets/viewer/assets/
```

Vite 构建输出的 HTML/JS/CSS 以 Flutter asset 方式打入应用包，通过 `rootBundle` 访问。

### 6.2 运行时提取

首次运行 `LocalServer.start()` 时，调用 `_extractViewerAssets()`：

1. 在系统**临时目录**下创建 `prop_os_cad_viewer/` 目录。
2. 检查版本戳文件 `.extracted`，内容与硬编码常量 `extractionVersion`（当前为 `'v2'`）匹配则跳过提取（缓存复用）。
3. 逐一通过 `rootBundle.load(assetPath)` 读取 asset 字节写入临时目录。
4. 写入版本戳文件，下次启动直接复用。

**版本冲突处理：** 升级前端构建产物时，只需将 `extractionVersion` 常量递增（如改为 `'v3'`），下次启动即自动触发重新提取。

| 平台 | WebView 后端 | 特殊说明 |
