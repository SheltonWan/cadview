## 1. 架构概述

PropOS CAD 采用 **"Flutter 原生壳 + 内嵌 Web 查看器"** 的混合架构，将两种技术的优势组合：

- **Flutter**：提供跨平台原生 UI、文件系统访问、应用生命周期管理。
- **Web（Vue + WebAssembly）**：提供高性能 CAD 图形解析与 2D/3D 渲染，复用成熟的 CAD 开源生态。

两者通过一个**应用内嵌 HTTP 服务（LocalServer）**和 **JavaScript Bridge** 协调通信，避免了直接 `file://` 协议带来的浏览器安全限制问题。

## 3. 层次结构说明

应用共分为四个层次：

### 第一层：Flutter UI 层

负责原生用户界面的渲染与交互。包含两个主要页面（Screen），通过 Flutter Navigator 栈进行页面跳转。

### 第二层：服务层

包含 `LocalServer` 单例服务，为 WebView 提供基于 HTTP 协议的本地资源访问能力。该层屏蔽了本地文件系统与 WebView 之间的协议差异。

### 第三层：Asset 层

将 CAD 查看器前端（由 Vite 构建）的产物打包为 Flutter 静态资源，随应用发布。运行时由 `LocalServer` 解压至临时目录并通过 HTTP 提供服务。

### 第四层：WebView / 前端层

运行于 WebView 沙箱中的 Vue 3 应用，负责 CAD 文件的解析（WebAssembly）和图形渲染（Canvas/WebGL）。通过 JS Bridge 与 Flutter 双向通信。

## 4. 核心模块设计

### 4.1 `PropOsCadApp`（应用根）

- 类型：`StatelessWidget`
- 职责：创建 `MaterialApp`，注入全局主题。
- 首页：`FileBrowserScreen`。

### 4.2 `FileBrowserScreen`（文件浏览）

- 类型：`StatefulWidget`
- 职责：
  - 调用系统文件选择器（`file_picker`）让用户选择目录或文件。
  - 扫描目录，过滤并展示 `.dwg` / `.dxf` 文件列表。
  - 响应文件点击，导航至 `ViewerScreen`。
- 状态：`_currentDir`、`_dwgFiles`、`_loading`。

```
用户点击 → FilePicker 对话框 → 获得路径 → listSync + filter → setState → ListView 渲染
```

### 4.3 `ViewerScreen`（CAD 查看）

- 类型：`StatefulWidget`
- 职责：
  - 确保 `LocalServer` 已启动。
  - 创建并持有 `WebViewController`，加载查看器页面。
  - 管理查看器就绪状态，控制加载蒙层的显隐。
  - 通过 `runJavaScript` 将文件 URL 传递给前端。
  - 通过 `FlutterBridge` 接收前端事件（ready / error）。
- 状态：`_controller`、`_viewerReady`、`_serverError`、`_errorMessage`。

### 4.4 `LocalServer`（本地 HTTP 服务）

- 类型：Dart 单例类
- 职责：
  - 解压 Flutter asset 至临时目录（带版本缓存）。
  - 监听 `127.0.0.1:0`，获取系统分配的空闲端口。
  - 处理 `/viewer/**` 路由（静态文件服务）。
  - 处理 `/dwg?path=` 路由（本地文件代理）。
  - 注入 CORS 头与 SharedArrayBuffer 所需安全头。
- 对外接口：`start()`、`stop()`、`port`、`isRunning`。

### 4.5 Vue CAD Viewer 前端

- 入口：`src/main.ts`
  - 挂载 Vue 应用，注册 Element Plus 及其图标。
  - 引入 vue-i18n。
- 核心组件：`src/App.vue`
  - 使用 `<MlCadViewer>` 组件，`:url` 绑定响应式 `fileUrl`。
  - `onMounted` 时挂载 `window.cadViewer` API 并发出 `ready` 事件。

## 5. 数据与控制流

### 5.1 启动流程

```
应用启动
  └─ WidgetsFlutterBinding.ensureInitialized()
  └─ PropOsCadApp → MaterialApp → FileBrowserScreen
       └─ 显示欢迎页（等待用户操作）
```

### 5.2 文件打开流程

```
用户点击"选择文件夹"
  └─ FilePicker.getDirectoryPath()
  └─ Directory.listSync() → filter .dwg/.dxf → sort
  └─ setState(_currentDir, _dwgFiles)
  └─ ListView 渲染文件列表
       └─ 用户点击某文件
            └─ Navigator.push(ViewerScreen(filePath))
```

### 5.3 CAD 文件加载流程

```
ViewerScreen.initState()
  └─ LocalServer.instance.start()
  │    └─ _extractViewerAssets()     ← 首次：解压 asset 至临时目录
  │    └─ shelf_io.serve(127.0.0.1:0)
  │
  └─ 构建 WebViewController
  └─ 加载 http://127.0.0.1:{port}/viewer/
       │
       ▼ WebView 发出 HTTP GET /viewer/index.html
  LocalServer._serveViewer() → 返回 index.html
       │
       ▼ 浏览器加载 index.js（Vue 应用启动）
  Vue App.onMounted()
       └─ 挂载 window.cadViewer
       └─ FlutterBridge.postMessage({ type: 'ready' })
            │
            ▼ Flutter _onBridgeMessage(type='ready')
       setState(_viewerReady = true)
       _loadFile()
            └─ runJavaScript("window.cadViewer.loadFile('http://127.0.0.1:{port}/dwg?path=<encoded>')")
                 │
                 ▼ WebView 发出 HTTP GET /dwg?path=<encoded>
            LocalServer._serveDwg() → 读取本地文件字节 → 返回
                 │
                 ▼ @mlightcad/cad-viewer 解析并渲染 DWG
            CAD 图纸显示 → 加载蒙层消失
```

### 5.4 错误处理流程

```
前端发生错误
  └─ FlutterBridge.postMessage({ type: 'error', payload: '错误描述' })
       └─ Flutter _onBridgeMessage(type='error')
            └─ ScaffoldMessenger.showSnackBar(红色提示)
```

## 7. 关键设计决策

### 7.1 为何使用内嵌 HTTP 服务而非 `file://`

**问题：** WebView 通过 `file://` 加载 HTML 时，`SharedArrayBuffer`（WebAssembly 多线程必需）会被浏览器安全策略（`Cross-Origin-Isolation` 要求）阻止；`file://` 协议加载的资源也无法使用 Web Worker fetch 其他本地文件（CORS 限制）。

**决策：** 在应用内部启动一个仅监听 `127.0.0.1` 的 HTTP 服务，所有资源通过 `http://127.0.0.1:{port}` 提供，配合 `COOP/COEP` 头启用跨域隔离，解除 SharedArrayBuffer 限制。

**权衡：** 增加了启动延迟（首次需要解压 asset），以及额外的 HTTP 请求往返开销，但使得功能可用性得到保证。

### 7.2 为何在 Asset 中打包前端构建产物

**问题：** 如果前端代码通过 `https://cdn.example.com` 或外部 URL 加载，应用将依赖网络连接，无法离线使用。

**决策：** 将 Vite 构建产物（HTML/JS/CSS/WASM Worker）以 Flutter asset 形式打包进应用，实现完全离线运行。

### 7.3 动态端口分配

**问题：** 若硬编码固定端口（如 8080），在用户设备上可能因端口被占用导致服务启动失败。

**决策：** 传入端口 `0`，由操作系统分配空闲端口，之后通过 `_server!.port` 获取实际端口号并共享给 WebView 使用。

### 7.4 Asset 提取版本戳

**问题：** 每次启动全量解压 asset 会增加不必要的 IO 开销。

**决策：** 在临时目录写入版本戳文件，版本字符串与代码中 `extractionVersion` 常量一致时直接复用缓存。升级前端产物时手动递增该常量即可触发重新提取。

## 8. 扩展点与演进建议

| 扩展方向 | 建议实现方式 |
|----------|-------------|
| 递归扫描子目录 | 将 `FileBrowserScreen` 中的 `listSync` 替换为递归实现，或使用目录树组件 |
| 最近打开文件记录 | 引入 `shared_preferences` 存储最近文件路径列表 |
| 搜索/筛选文件 | 在文件列表上方添加搜索框，对 `_dwgFiles` 进行本地过滤 |
| 图纸内容导出（截图） | 通过 `window.cadViewer.exportImage()` 扩展 JS Bridge API |
| 图层管理面板 | 在 Vue 前端通过 `@mlightcad/cad-viewer` 的图层 API 添加侧边面板 |
| 状态管理扩展 | 目前 `provider` 已作为依赖引入，可用于跨页面共享文件选择状态 |
| 多文件标签页 | 在 `ViewerScreen` 层增加 Tab 管理，每个 Tab 维护独立 `WebViewController` |
